# 配置 GitHub 默认邮件通知

本文档介绍如何设置 GitHub 默认的邮件通知，以便在多语言新闻发布系统工作流执行完成时收到通知。

## GitHub 默认邮件通知概述

GitHub 提供内置的邮件通知功能，可以在工作流执行成功、失败或其他重要事件发生时通知仓库的相关人员。这种方式不需要配置额外的邮件服务器或密钥，而是利用 GitHub 自身的通知系统。

## 设置步骤

### 1. 配置个人邮件通知首选项

每个需要接收通知的用户都需要进行以下设置：

1. 登录 GitHub 账户
2. 点击右上角头像，选择 "Settings"（设置）
3. 在左侧菜单中选择 "Notifications"（通知）
4. 在 "Email notifications"（邮件通知）部分：
   - 确保已添加并验证您的电子邮件地址
   - 选择 "Participating and @mentions"（参与和@提及）或更广泛的通知范围
   - 对于工作流执行的通知，请确保选中 "Actions" 相关选项

### 2. 关注仓库

为确保您能收到通知，您应该：

1. 访问项目仓库
2. 点击右上角的 "Watch"（关注）按钮
3. 选择适当的通知级别：
   - "All Activity"（所有活动）
   - "Participating and @mentions"（参与和@提及）
   - "Custom"（自定义）- 可以专门选择接收 Actions 工作流通知

### 3. 工作流设置（已完成）

我们已经在工作流配置中添加了通知步骤，通过使用：

- `$GITHUB_STEP_SUMMARY` 生成工作流摘要，显示在工作流运行详情页面
- GitHub 的通知命令（`::notice::`, `::warning::`, `::error::`）来触发适当级别的通知

## 通知类型说明

您将收到的通知取决于工作流执行结果：

- **成功通知** - 当发布流程和站点地图更新全部成功时
- **警告通知** - 当部分任务成功，部分任务失败或跳过时
- **错误通知** - 当关键任务失败时

## 调整通知频率

如果您收到过多的通知，可以调整通知设置：

1. 前往您的 GitHub 通知设置
2. 选择适当的通知频率：
   - "Email each time"（每次发送邮件）
   - "Email daily digest"（每日摘要）
   - "Email weekly digest"（每周摘要）

## 故障排查

如果您没有收到预期的通知：

1. 检查您的邮件垃圾箱
2. 确认您已正确设置 GitHub 通知首选项
3. 验证您的邮箱是否已在 GitHub 上验证
4. 确认您是否已关注相关仓库
5. 检查仓库的权限设置，确保您有足够的访问权限收到通知

## 与仓库管理员相关的设置

作为仓库管理员，您可以：

1. 在仓库设置中邀请协作者，以便他们可以接收通知
2. 为组织配置默认通知设置
3. 确保工作流有适当的权限设置来发送通知
